Overview

このツールは Guitar Hero / Fap Hero スタイルの譜面を作って再生する ための2画面構成の Web アプリです。 Editor で譜面を作り、Player で動画・音声と同期させて再生します。 譜面は JSON ファイルで管理されます。

作業フロー

  1. 1
    音声・動画を用意する
    再生したい音声ファイル（mp3/wav 等）と、オーバーレイする動画ファイル（mp4 等）をローカルに用意します。動画は必須ではありません。
  2. 2
    Editor で BPM と Offset を設定する
    曲の BPM と Offset を入力します。Offset の意味は下の「Offset とは」セクションを参照してください。Grid（分解能）は 16分音符（×4）がおすすめです。
  3. 3
    パターンを作成する
    左パネル「Patterns」で + New Pattern を押してパターンを追加。名前を付け、ステップシーケンサーでノーツを打ち込みます。1パターンは8拍（デフォルト）のループ単位です。
  4. 4
    タイムラインに並べる
    パターンカードを右パネル上部の Timeline にドラッグ＆ドロップして配置します。ブロック右の ×N を変えると繰り返し数を指定できます。ブロック同士もドラッグで並び替え可能です。
  5. 5
    プレビューで確認する
    画面下部の Preview でノーツの流れを確認します。Space で再生/一時停止、, で先頭に戻ります。メトロノームの小節頭クリック（高音）でリズムの合わせやすさを確認しましょう。
  6. 6
    JSON をエクスポートする
    右上の Export JSON で譜面ファイルを保存します。
  7. 7
    Player で再生する
    Player 画面で 譜面 JSON・音声ファイル・動画ファイル を読み込みます。Space で再生、Ctrl+Enter でシアターモードに切り替えるとノーツと動画だけの全画面表示になります。

ダイナミック編集モード（新機能）

Edit Mode を使用すると、リズムに合わせてスペースキーを叩くだけでリアルタイムでビートを記録できます。BPM や Offset を手動で入力する必要がありません。

  1. 1
    音声・動画を読み込む
    Player 画面で音声ファイル（とオプションで動画）を読み込みます。
  2. 2
    Edit Mode を有効にする
    🎯 Edit Mode ボタンをクリックします。ボタンがシアン色に変わり、「Edit Mode ON」と表示されます。
  3. 3
    再生してビートを記録
    ▶ Play をクリックし、リズムに合わせて Space キーを叩きます。記録されたビートはシアン色のノートとしてリアルタイムで表示されます。
  4. 4
    一時停止と再開
    ⏸ Pause で一時停止すると、再開時に5秒戻ってカウントダウンが表示されます。カウントダウン中のスペースキー入力は無視されるため、リズムを再同期する時間があります。
  5. 5
    既存の譜面を続ける
    既存の JSON 譜面を読み込んでから Edit Mode を有効にすると、その譜面のビートを引き継いで記録を続けることができます。
  6. 6
    エクスポート
    💾 Export をクリックして、記録したビートマップを JSON ファイルとして保存します。BPM は自動的に計算されます。
  7. 7
    再生して確認
    Edit Mode を無効にして、作成したビートマップを再生して確認します。

ヒント： 記録中に 🗑️ Clear ボタンをクリックすると、ビートマップをリセットできます（音声・動画は保持されます）。

Offset とは

Offset（ms） は「音声ファイルの先頭から、最初の1拍目が来るまでの時間（ミリ秒）」です。

例： 音声ファイルの最初の 2 秒間がイントロや無音で、2 秒後に 1 拍目（ビート）が始まる場合 → Offset = 2000

逆に音声がいきなり 1 拍目から始まる場合は Offset = 0（デフォルト）のままでOKです。

Offset がズレていると、ノーツと音楽のリズムが噛み合わなくなります。 曲を聴きながらメトロノームのクリック音（プレビュー再生）がビートと一致するよう微調整してください。 10ms 単位 で詰めると合わせやすいです。

BPM や Offset を変更すると既存ノーツの実時間上の位置が変わります。譜面を打ち込む前に確定させるのを推奨します。

Tips

BPM が分からないとき

BPM 検出ツール（例: Tap BPM 系の Web ツール）で曲に合わせてタップすると大体の BPM が得られます。その後 Offset でズレを微修正するのが効率的です。

パターンの使い方

「Aメロ」「サビ」「ブリッジ」のようにセクション単位でパターンを分けると、タイムライン上での管理が楽になります。同じフレーズはパターンを繰り返し（×N）で使い回せます。

グリッド変更の注意

Grid（分解能）を変更するとすべてのパターンのノーツがクリアされます。譜面を打ち込み始める前に決めてください。

譜面 JSON のバックアップ

ブラウザはデータを保存しません。こまめに Export JSON でファイルに保存してください。次回は Import JSON で読み込めます。

Edit Mode の活用（新機能）

Edit Mode は BPM や Offset が分からない場合に最適です。リズムに合わせてスペースキーを叩くだけで、BPM が自動的に計算されます。既存の譜面を Edit Mode で開くと、その譜面の続きから記録を続けることができます。

動画速度の調整（新機能）

Video Speed スライダーで動画の再生速度を 0.1x から 4.0x まで調整できます。PMV 制作時に、曲の速い部分で動画を加速したり、好きなシーンでスローモーションにしたりと、リアルタイムで編集できます。

動画ループ（新機能）

動画が音声より短い場合、自動的にループ再生されます。短いループ動画を背景に使って、長い曲全体で再生できます。

自動保存（新機能）

Edit Mode で記録中のデータは localStorage に自動保存されます。ブラウザを閉じても、次回 Edit Mode を有効にすると記録を続けることができます。
